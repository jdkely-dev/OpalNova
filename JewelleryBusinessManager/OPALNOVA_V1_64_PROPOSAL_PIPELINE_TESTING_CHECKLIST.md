# OPALNOVA V1.64.0 Proposal Pipeline Testing Checklist

Use this checklist after launching the V1.64.0 build.

- [ ] Confirm the main header shows `Version 1.64.0 - proposal pipeline`.
- [ ] Open About and confirm it shows `Version 1.64.0 - Proposal Pipeline`.
- [ ] Open Workflow > Proposal Pipeline.
- [ ] Confirm the pipeline opens as a full workspace tab and uses the dark OPALNOVA theme.
- [ ] Confirm filters work for Action needed, Prepared not sent, Follow-up due, Sent, Accepted, Converted and All proposals.
- [ ] Type part of a quote code, customer name, title or email into Search and confirm the grid narrows.
- [ ] Select a row and confirm the right-side details show proposal status, quote status, dates, recipient, option and proposal file state.
- [ ] Use Open Quote and confirm the selected quote opens in the quote builder, not just the generic quote list.
- [ ] If the selected quote has a generated proposal file, use Open Proposal File and confirm the file opens.
- [ ] If the selected quote has a recorded email draft, use Copy Email Draft and confirm clipboard content is ready to paste into email.
- [ ] Use Create Follow-Up on one proposal and confirm a task is created once.
- [ ] Run Create Follow-Up again for the same proposal and confirm OPALNOVA prevents a duplicate open task.
- [ ] Close the Proposal Pipeline tab with its tab close button and with the bottom Close button.
- [ ] Reopen Project Workbench, Alert Centre, Quotes & Proposals, Reports and Custom Workflow Studio and confirm Proposal Pipeline actions are available where expected.
- [ ] Close the app normally.
