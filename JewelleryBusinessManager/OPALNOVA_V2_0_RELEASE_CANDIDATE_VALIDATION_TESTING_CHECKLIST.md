# OPALNOVA V2.0.0 Release Candidate Validation Testing Checklist

Use this checklist against the published V2.0 build before treating it as a whole-number milestone checkpoint.

- Launch OPALNOVA and confirm the header shows `Version 2.0.0 - release candidate validation`.
- Open About and confirm it shows `Version 2.0.0 - Release Candidate Validation`.
- Open Settings & Backup and run `Release Notes`; confirm V2.0.0 appears above V1.99.0.
- Open Settings & Backup and run `User Guide`; confirm the manual version shows V2.0.0.
- Open Settings & Backup and run `Support Snapshot`; confirm installed version context reports the current 2.0 build.
- Open Safety & Data Studio and run `Release Readiness`; confirm the report opens in preview.
- Open Customer Relationship Studio and confirm `Customer Timeline` mini help opens the Customer Timeline-specific guide.
- Open Customer Relationship Studio and confirm `Communication Templates` mini help opens the Communication Templates guide.
- Reopen the V1.91-V1.99 changed workflows and support reports: Nivoda Staging Handoff, Operations Performance, Inventory Intelligence, Payment & Collection, Production Board and Support Snapshot.
- Confirm selector fields show friendly prompts instead of raw object strings or blank white faces.
- Confirm no new action creates duplicate tasks, payments, stock movements, sales or supplier diamond state changes.
- Confirm Debug build, Release publish and published launch smoke have passed before treating this as release-ready.
