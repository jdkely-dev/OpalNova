# OPALNOVA Backup and Restore Guide

## Before first real use

Create a clean backup before entering important business data.

Recommended backup name:

```text
OPALNOVA_clean_start_backup
```

## Daily/weekly backup habit

For active business use, create a backup:

- before major edits
- after market days
- after entering a batch of sales
- before testing a new build
- weekly during normal use

Store backups somewhere outside the app folder, such as:

- an external drive
- a cloud folder
- a dedicated business backup folder

## Before updating OPALNOVA

1. Open the current working version.
2. Create a backup.
3. Close the app.
4. Copy the backup somewhere safe.
5. Only then test the new version.

## Restore safety

Only restore from a backup you trust. If restoring because something looks wrong, copy the current data folder first so you have a fallback.
