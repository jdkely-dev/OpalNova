# OPALNOVA V1.77.0 Recent Work Recall Testing Checklist

Use this checklist after launching the V1.77.0 build.

- [ ] Confirm the main header shows `Version 1.77.0 - recent work recall`.
- [ ] Open About and confirm it shows `Version 1.77.0 - Recent Work Recall`.
- [ ] On the dashboard, confirm the `Recent Work` panel starts with the empty-state message.
- [ ] Open Production Board and return to the dashboard.
- [ ] Confirm `Production Board` appears in Recent Work.
- [ ] Click the Production Board recent entry and confirm the workflow tab opens or switches to the existing tab.
- [ ] Generate a report, such as Release Notes or Business Report.
- [ ] Return to the dashboard and confirm the generated report appears in Recent Work.
- [ ] Click the report recent entry and confirm the report preview reopens.
- [ ] Open a saved record editor, such as an existing job or customer.
- [ ] Return to the dashboard and confirm the saved record editor appears in Recent Work.
- [ ] Click the record recent entry and confirm the editor reopens for the same record.
- [ ] Change a field in the reopened editor and try closing the tab.
- [ ] Confirm Save, Discard and Cancel protection still appears for changed hosted editor tabs.
- [ ] Open the same workflow/report/record more than once and confirm Recent Work does not create duplicates.
- [ ] Click `Clear` and confirm the recent list returns to the empty-state message.
- [ ] Confirm no records are created or changed just by opening Recent Work entries.
